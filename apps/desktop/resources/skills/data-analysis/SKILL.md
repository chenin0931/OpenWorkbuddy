# 数据分析

把附件数据转化为可复现、经过验证且能在产物区直接打开的分析结果。

## 工作流

1. 从任务附件清单取得 `artifactId`，调用 `attachment_open`。不得按附件名扫描桌面、主目录或磁盘。
2. 检查文件结构、字段类型、缺失值、单位、时间范围、标签含义和潜在泄漏。若关键字段语义无法从数据与用户目标判断，先说明具体歧义再询问。
3. 在工作区内建立任务私有环境，例如 `.openworkbuddy/runs/<run-id>/venv`。不得向用户全局 Python 安装依赖，不得修改系统 Python。
4. 先确定最小分析方案和验收指标，再处理数据。统计、模型和图表必须由同一份最终脚本完整生成；临时命令只用于探索，不能成为唯一实现。
5. 预测或分类模型至少使用留出集或交叉验证。报告训练集与验证集口径、基线、指标和限制，禁止只报告训练集效果。
6. 从干净输出目录运行最终脚本，确认它能重新生成全部统计、图表和报告。保存命令、退出码和关键校验结果作为证据。
7. 使用 `output_register` 登记最终脚本、报告、数据表和图表。未验证、估算或无法复现的部分必须在最终回答中明确列出。

## 完成标准

- 数据来源与字段语义明确，未知项没有被静默猜测。
- 最终脚本从输入附件可完整复现结果，不依赖交互式会话或全局环境。
- 模型评估包含留出集或交叉验证，指标与数据切分可追溯。
- 所有交付文件已登记到产物区，并完成一次干净目录复现验证。
