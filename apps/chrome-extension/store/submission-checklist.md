# Chrome Web Store submission checklist

- [ ] Create the publisher account and verify its email.
- [ ] Confirm the published GitHub privacy-policy URL is accepted by Chrome Web Store.
- [ ] Upload `on-my-workbuddy-chrome.zip`.
- [ ] Use the single-purpose and permission explanations from `listing-zh-CN.md`.
- [ ] Add 1280×800 or 640×400 screenshots showing the user-click binding flow.
- [ ] Confirm the production Native Messaging manifest uses the final Web Store extension ID.
- [ ] Complete data-use disclosures accurately: no developer cloud, sale, advertising or telemetry; authorized page data may be sent by the desktop app to the user's configured model provider.
- [ ] Run the tab-isolation, debugger-detach and native-host reconnect tests against the store build.
