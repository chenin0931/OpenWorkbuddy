# Changelog

## Unreleased

- 将全部公开品牌文案统一为 `OpenWorkbuddy`，覆盖桌面应用、Chrome Bridge、MCP OAuth、通知、文档与测试。
- 首次启动新版时自动迁移预览版的 macOS Application Support 数据目录，并保留 Native Messaging Host 兼容链接。

## 0.3.0 — 2026-07-12

- 将桌面端重构为 “Quiet Workbench / 安静的工作台”：统一暖灰、石墨与深钴蓝设计 Token、排版、间距、圆角和浮层阴影，恢复原生 macOS 标题栏。
- 新增 `WorkTurnViewModel` 视图聚合层；同一次请求只呈现一个用户块和一个 WorkBuddy 响应，多次工具调用归并为默认收起的单行活动摘要。
- 默认采用侧栏与工作内容两栏结构，详细、变更和活动面板按需打开；无内容区块、普通完成状态与内部工具诊断不再占据主界面。
- 重写审批、错误与结果语言：只突出需要用户处理的状态，将检查、Diff、来源和输出作为紧邻回答的结果证据呈现。
- 建立统一 Phosphor 功能图标适配层与新的折叠工作面品牌标识，并补齐键盘焦点、高对比度、Reduce Motion、浅深色和固定场景视觉回归。
- 本次只重构 Renderer 视图层与产品语言；Agent Host、SQLite 状态与恢复、工具权限、审批执行和安全边界保持不变。

## 0.2.6 — 2026-07-12

- 普通对话结束后不再显示“完成 / 部分完成”徽标、侧栏圆点或常驻结果卡；只有运行中、等待批准、等待回复、暂停和失败等需要关注的状态留在主界面。
- 普通模型回复不再自动伪造 `partial` 结论；只有真实执行 `task_complete` 并生成核验回执后，右侧检查器才显示“已验证 / 待核验”。
- 普通问答、寒暄和“结束任务”等确认不再调用完成门禁；即使旧模型误调 `task_complete`，没有本轮操作回执也不会生成或展示“待核验”。
- 为每次续聊记录独立的 `run.turn_started` 边界，清除上一轮摘要、错误与结果，并将完成门禁限定到当前轮工具、步骤和产物。
- 统一生产状态迁移入口，终态续聊必须显式开启新一轮；暂停、取消和 Chrome 断线会使未完成审批与工具回执失效。
- 修复拒绝审批后工具回执悬空、暂停或取消时 Shell 子进程未按任务取消，以及历史验证或计划串入新追问的问题。
- 增加普通对话、可操作状态、当前轮核验、终态续聊、预算耗尽和状态迁移回归测试。

## 0.2.5 — 2026-07-12

- 新增用户消息边界上的 Agent 回合容器：同一请求中的前置说明、工具调用和最终回复只显示一次头像与标题。
- 工具过程继续默认收起，并嵌入对应 Agent 回合；下一条用户消息才开始新的 Agent 会话块。
- 修复模型工具调用 ID 被误作全库主键的问题，使用独立 UUID 回执并保留原始 Provider Call ID。
- 启动时自动迁移旧数据库，避免续聊或跨任务重复出现 `UNIQUE constraint failed: tool_calls.id`。
- 增加多段回复/多工具回合合并、重复 Provider Call ID、旧表迁移及审批映射回归测试。

## 0.2.4 — 2026-07-12

- 工具与执行过程默认收起，运行中的最新工具组也不再自动展开。
- 收起态压缩为单行摘要，点击后才展示工具参数、结果与时间，再次点击即可收回。
- 增加运行中工具组“默认隐藏 → 手动展开 → 再次收起”的界面回归测试。

## 0.2.3 — 2026-07-12

- 用户追问在发送请求返回前立即显示为右侧气泡，并在持久化回读后无缝替换，避免延迟或重复。
- 消息、工具调用按真实时间顺序交错展示；连续工具调用分段折叠，后续追问不再被长活动记录遮住。
- 增加“即时回显、右侧对齐、工具在前、刷新不重复”的端到端回归测试。

## 0.2.2 — 2026-07-11

- 修复纯工具回合被保存成空白助手消息的问题，同时保留模型 usage 审计。
- 使用安全的 Markdown + GFM 渲染对话内容，阻止原始 HTML 与非 HTTP(S) 外链。
- 将生命周期与完成结果组合展示，`completed + partial` 统一显示为琥珀色“部分完成”。
- 修复长内容撑破任务 Grid 的问题，让输入框固定在视口内；已完成或失败任务可继续追问。
- 新增持久化工具回执、网页证据和审批历史；应用重启后仍可审查 Search/Fetch 来源。
- 重构右侧检查器为概览、证据、活动和产物，明确区分“搜索发现”“已读取原文”和“已验证”。
- 按页面声明的 charset 解码 GBK/GB2312 等中文网页，并阻止网页正文或大型预览进入 Renderer 公共投影。
- 统一 `OpenWorkbuddy` 品牌文案，改善次要文字对比度和窄窗口布局。

## 0.2.1 — 2026-07-11

- 新增 Provider 无关的 `web_search`，通过固定 Bing Web 结果页返回受限的标题、URL 与摘要。
- 搜索首次执行前显示离机数据审批，可按本任务授权；只读研究型子 Agent 获批后也可使用。
- 搜索与 `web_fetch` 共用 DNS pin、私网/保留地址阻断、逐跳重定向校验和响应上限。
- 搜索结果继续按不可信工具数据处理；重要结论需再用 `web_fetch` 打开原文核验。
- 将 Search/Fetch 提升为核心可见工具，并用 Pi `prepareNextTurnWithContext` 修复运行中渐进加载工具未进入后续回合的问题。
- x64 打包前显式重编 Electron ABI 的 `better-sqlite3`，避免 Node 单测与 Electron 发行包之间的原生模块串用。
- 增加搜索参数、审批、固定主机、SSRF、重定向、解析漂移与在线后端冒烟测试。

## 0.2.0 — 2026-07-11

- 新增独立的 Kimi / Moonshot 中国区 Provider，默认模型 `kimi-k2.7-code`。
- 固定 Chat Completions 地址和 K2.7 参数边界：256K 上下文、32K 产品输出上限、仅思考模式、稳定 `prompt_cache_key`。
- 多轮工具调用保留 `reasoning_content`，禁用不支持的 `tool_choice=required` 和 Responses/旧 functions 参数。
- 增加旧 SQLite `model_profiles` Provider CHECK 约束的无损迁移。
- 设置与首次启动流程增加 Kimi，并强化 Key 状态清理、公开错误脱敏与重试分类。
- 增加 Kimi 离线协议测试、假 Key UI E2E、隐藏输入在线工具调用和完整 Electron safeStorage 联调。

## 0.1.0 — 2026-07-10

- 首个本地 x64 交付：任务工作台、文件/Shell、MCP、Skills、Memory、Chrome Bridge、自动化、审批、验证与恢复。
