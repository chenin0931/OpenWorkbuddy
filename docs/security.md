# Security model

## 能保证的边界

- Renderer 不直接访问 Node、文件、Key 或原始 IPC。
- 文件工具在任务授权根目录内执行 realpath 和符号链接检查。默认根为当前工作区；只有用户在输入框明确选择“完全访问”后，该任务的根才变为 `/`。
- 项目工作区与授权根分离：全盘模式下相对路径和 Shell 默认 cwd 仍落在项目目录，绝对路径才可跨工作区；权限随 Run 持久化并由子 Agent 继承。
- 模型 Key 使用系统加密存储，工具进程不会继承这些变量。
- 请求批准模式下，外部副作用与不可逆动作必须审批；完全访问模式下，所有未被确定性策略硬拒绝的工具会自动执行。
- 执行权限由用户在每项工作的输入框中选择，并可为新工作保存默认值。完全访问同时放宽文件授权根和交互审批，但不能绕过 macOS TCC、只读子 Agent、参数 Schema、路径真实化、陈旧写入保护或产品硬拒绝。
- 工具调用和审批有本地审计记录。

## 不能保证的边界

- 本地 Shell 不是恶意代码沙箱；获批命令或完全访问模式自动执行的命令都以当前 macOS 用户运行。
- “完全访问”无法绕过 macOS TCC；没有系统 Full Disk Access 时，受保护目录仍可能返回 `EPERM`。
- 用户安装的 stdio MCP server 是本机代码，产品无法阻止它绕过 Agent 的文件工具。
- Chrome `debugger` 权限能力很强；产品按任务约束标签页，但 Chrome 扩展权限本身并非细粒度 OS 隔离。
